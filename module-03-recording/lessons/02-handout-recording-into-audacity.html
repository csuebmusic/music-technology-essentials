Module 03 · Recording, sample prep & library building

Recording into Audacity

Capture four sounds, prep them into samples, start your library.

Monday's reading walked you through the basic recording chain on paper. Today you plug it in. By the end of this session you'll have set up Audacity for recording, dialed in a clean level on the interface, captured four short recordings (paper crumble at two speeds, paper rip at two speeds), prepped each one into a usable sample, and organized the lot into a folder structure that's the start of a sample library you'll grow for the rest of the semester. The work today is the foundation for every recording session that follows.

Today's gear

Take from the lab's gear storage: audio interface, headphones, dynamic mic (with stand and XLR cable). Plug in and run through the start-of-session steps on the laminated Session Routines card before continuing here.

What we're doing today

  1. Set up your sample library folder.
  2. Configure Audacity for recording: project format, recording device, mono track.
  3. Set the gain knob on the interface using Audacity's input meter. Aim for peaks at -12 to -6 dBFS.
  4. Set the monitor blend so you can hear yourself while recording.
  5. Do a test recording. Listen back. Adjust if needed.
  6. Capture a short noise-profile clip (room tone) for the denoise step later.
  7. Record the four paper sounds.
  8. Prep each recording through the three-step pipeline: denoise → trim → normalize.
  9. Export each prepped sample as a WAV.
  10. Write the library README. Upload everything to the NAS.
First time with the mic

The four-stage chain you read about Monday (mic → XLR → interface → computer) is now physically in front of you. If anything looks wrong, or a cable won't plug in cleanly, raise your hand before forcing anything.

Step 1 · Set up your sample library folder

Build the folder now, before you open Audacity. This is the home for every sample you record this semester; you'll be living in it for the rest of the course, so take a minute to get the structure right.

  1. In Finder, navigate to your local working folder: ~/Documents/lastname/. (If you don't have a folder there yet, make one with your lowercase last name.)
  2. Inside that folder, create a new folder named sample-library. Lowercase, hyphenated. This is the library's root.
  3. Inside sample-library/, create a folder named paper. This is the first category folder; you'll add more categories (metal, water, voice, etc.) as you record more sounds in later weeks.
  4. Inside sample-library/, also create a folder named audacity-projects. This is where your working Audacity project file will live (you'll save it there in a few minutes). Keeping it separate from the prepped samples means you can browse paper/ as a flat list of finished sounds without your .aup3 file cluttering the view.

Your folder should now look like this:

~/Documents/lastname/
  sample-library/
    paper/                    <-- empty for now; samples land here in Step 9
    audacity-projects/        <-- empty for now; the .aup3 project lands here after the test recording

Step 2 · Configure Audacity for recording

Open Audacity. Before recording, you need to check the audio settings.

Check the project format in Preferences

  1. Open Audacity → Preferences… (or Cmd + ,). The Preferences window opens.
  2. In the left sidebar, click Audio Settings.
  3. Confirm (or set) the Module 2 standards: Project Sample Rate at 44100 Hz, Default Sample Format at 16-bit PCM.
  4. Click OK to close the Preferences window.
The recording device lives here too

The Audio Settings panel in Preferences also has Recording Device, Playback Device, and Recording Channels dropdowns. You can change them here. The next section shows you the same change from the main window's Audio Setup button, which is what you'll use day to day; the Preferences panel is just where everything lives if you ever need to dig in.

Select the audio interface as your recording device

  1. At the top of the Audacity window, find the Audio Setup button (marker 5 on the empty-project tour from your orientation lab).
  2. Click it. A dropdown opens with several sections: Host, Playback Device, Recording Device, Recording Channels.
  3. Under Recording Device, choose your audio interface (it will be listed by model name: USB Audio CODEC, PreSonus AudioBox USB 96, or U-PHORIA UM2 depending on which interface you took from storage).
  4. Under Recording Channels, choose 1 (Mono). The dynamic mic on input 1 is a single source, so you want a single-channel recording. Stereo recording is for setups with two mics; today is one mic, one channel.
  5. Under Playback Device, the interface should already be selected (so playback goes out the interface's headphone jack into your headphones). If it isn't, select it now.

Add a mono recording track

  1. From the menu bar, click Tracks → Add New → Mono Track.
  2. An empty mono track appears in the project. This is where your recording will land.
Why mono?

A dynamic mic plugged into one input on the interface is a mono source: one signal, one stream of numbers, one channel. Recording it to a mono track stores it correctly (one channel of audio) and produces a file that's half the size of an unnecessary stereo version. Stereo is for setups with two mics or a stereo source; today is one mic.

Step 3 · Set the gain on the interface

The gain knob on the audio interface controls how much the preamp amplifies the mic-level signal before the ADC converts it to digital. You met it in Monday's reading. Today you set it. The target you're aiming for: when you make the loudest version of the sound you're about to record, the peaks on Audacity's input meter land in the -12 to -6 dBFS band. That's the window; anything outside it means you need to adjust.

Three bands matter:

  • Safe (below -12 dBFS): the signal is below the target. The recording will be quiet and you'll need to amplify it later, which also amplifies the room noise. Turn the gain up.
  • Target (-12 to -6 dBFS): the peaks of your loudest sound sit here. You have headroom (room between your loudest moment and the digital ceiling), and the signal is well above the noise floor. This is where you want to be.
  • Ceiling (-6 to 0 dBFS): the signal is approaching the ceiling and you're close to clipping. If the clip light on the interface flashes, you've already crossed it. Turn the gain down.

To dial it in:

  1. Turn on live monitoring so the meter shows levels before you start recording. Click Audacity's recording level meter at the top of the window. (It's the meter on the right side of the top toolbar, with a small mic icon.) A small menu appears with two items: Enable Silent Monitoring and Options…. Click Enable Silent Monitoring. The meter goes live: it shows the signal coming in from the mic in real time, even though you're not recording yet. (The "silent" in the name means the audio isn't routed through Audacity to your headphones, only displayed on the meter; you'll still hear the mic through the interface's monitor blend, set in the next step.)
  2. Make a sound at the loudness you'll be aiming for. For the paper sounds, that means a sample crumble or a quick test rip at roughly the speed and intensity you'll use for the real take. Grab a small piece of scrap paper from the stack at your station for this; don't burn through the four sheets you'll need for the actual recordings.
  3. Watch the meter. If the peaks sit below -12, turn the gain knob on the interface clockwise (up) a quarter-turn and try again. If the peaks hit the red zone or trip the interface's clip indicator, turn the gain counter-clockwise (down).
  4. Repeat until the loudest peaks land in the -12 to -6 band consistently. Don't chase a single peak; you're aiming for where the loud moments settle, not the absolute maximum of any one try.
Why the monitoring step exists

Audacity's default is to keep the input meter dark unless the project is actively recording. To set the gain you need to see live levels before you press record, which is what Enable Silent Monitoring does. Once it's on, the menu item flips to Disable Silent Monitoring (so you can turn it back off if you ever need to). For setting gain, leave it on.

Don't normalize to fix a quiet recording

You might be tempted to undershoot the gain ("better safe than clipping") and plan to fix it in editing. Don't. A signal recorded too quietly carries less information than one recorded at the right level, and when you amplify it after the fact you amplify the room noise along with the sound. Set the gain right at the source. Normalize, in the prep step later, is for standardizing across already-good recordings, not for rescuing too-quiet ones.

Step 4 · Set the monitor blend

You need to hear yourself through the headphones while you're recording. The interface's monitor blend control decides what you hear: the live signal coming straight from the mic (no latency), the signal coming back from the computer (a few milliseconds late), or a mix of both.

Today you want to hear the mic directly, with little or no playback from the computer.

  • On a PreSonus AudioBox USB 96: turn the Mixer knob (between the two gain knobs and the main output) toward Inputs. Not all the way; about three-quarters of the way toward Inputs is a good starting point. You'll hear yourself in the headphones immediately.
  • On a Behringer U-Phoria UM2: press the Direct Monitor button in (the square button next to the input jacks). The button stays depressed. You're now hearing the live input.
  • If you can't hear yourself at all: check the headphone volume knob on the interface, then check the in-line slider on the headphone cable, then check the monitor blend is set as above. The headphones plug into the front of the interface, not the computer.

Speak into the mic. You should hear yourself in the headphones with no perceptible delay. If you hear a noticeable echo or stutter, the monitor blend is set toward Playback rather than Inputs; adjust it.

Step 5 · Do a test recording

Before you commit to the four real recordings, do one practice take to confirm the chain is working end to end.

  1. Make sure the empty mono track from Step 2 is in the project. If it isn't, add one (Tracks → Add New → Mono Track).
  2. Click the red Record button in the transport toolbar (or press R).
  3. Wait one second, then speak a short phrase ("test recording, one, two, three").
  4. Click Stop (or press Space).
  5. A waveform appears on the mono track. Click Skip to Start (Cmd + Home), then Play (Space). Listen.

Three things to check as you listen:

  • Volume. The recording should be clearly audible. If it's silent or very quiet, the gain may be too low (back to Step 3), or the recording device may not be set right (back to Step 2).
  • Cleanliness. The sound should be clean, with no crackling or harshness. If you hear distortion, the gain is too high. Turn it down (Step 3) and try again.
  • Background. A faint room hiss is normal; a strong hum usually means the XLR cable is loose at one end. Push the cable in firmly at the mic and at the interface.

When the test sounds clean, delete the test clip. Click the X in the track header (or select the track and press Cmd + Shift + K to remove the track) and add a fresh mono track. You want the project to start clean.

Save your work

Now is the right moment to save the Audacity project, before you start capturing real recordings. File → Save Project As…, navigate to ~/Documents/lastname/sample-library/audacity-projects/ (the folder you built in Step 1), name the project sample-library.aup3, click Save. From this point on, save every few minutes with Cmd + S.

Step 6 · Capture a noise-profile clip

Every room has a noise floor: the small constant level of background sound from the ventilation, the computers, the building. You can't hear it most of the time because your ears tune it out, but a microphone catches it, and a quiet recording will pick it up alongside the sound you actually want. Audacity has a Noise Reduction effect that can subtract it back out, but to use it the effect needs to see a clean sample of just the noise, with no other sound mixed in. That sample is called a noise profile, and you're going to record it now.

  1. Stand still and quiet at your station. Don't touch the mic, the cable, or the table.
  2. Click Record.
  3. Wait 10 seconds. Stay quiet. Don't move. Let the room speak.
  4. Click Stop.

You now have a 10-second clip of pure room tone on your track. This is your noise-profile clip. The waveform looks almost flat: just the small wiggles of the room noise itself, with no other content. Leave it where it is; you'll use it during the prep step.

Why a separate clip and not the start of each recording?

You could capture the noise profile at the start of every recording. Recording it once as its own clip is cleaner: you have one definitive sample of room tone, you set the noise profile from it once, then apply Noise Reduction to each of the four sound clips. Less to remember during recording, less to trim during prep.

Step 7 · Record the four paper sounds

Pick up a sheet of paper from the stack at your station. You're going to record four short sounds, each as its own clip on the same mono track.

The four recordings, in order:

  1. Paper crumble, slow. Take the sheet of paper. Crumble it slowly into a ball over about 8 to 10 seconds. Let the small individual crackles and creases be audible.
  2. Paper crumble, fast. Take a new sheet. Crumble it as quickly as you can, in about 2 to 3 seconds. A burst of sound rather than a gradual one.
  3. Paper rip, slow. Take a new sheet. Rip it slowly down the middle over about 6 to 8 seconds. Let the tearing be audible the whole way through.
  4. Paper rip, fast. Take a new sheet. Rip it as fast as you can, in one quick motion.

For each recording:

  1. Hold the paper about 15 to 20 centimeters in front of the mic (a hand's width away). Not closer; if you get too close, the proximity effect will exaggerate the lows and the sound will feel boomy.
  2. Click Record.
  3. Wait one full second of silence, then make the sound.
  4. After the sound ends, wait one full second more, then click Stop.

The one-second buffer at each end gives you room to trim cleanly in the prep step. If you cut the silence too tight at the recording stage, you risk clipping off the very beginning or end of the sound itself.

Between recordings, listen back. Click on the clip you just made, press Space, hear it. If a recording didn't capture what you wanted, click the clip's X to delete it, and try that one again. Don't move on until each recording sounds like what you intended.

Step 8 · Run the prep pipeline on each recording

You have five clips on one track: a noise-profile clip first, then the four paper sounds. The prep pipeline turns each of the four sound clips into a sample: a clean, standardized file ready to live in the library and be used in later projects. The pipeline has three steps, applied in order: denoise → trim → normalize.

You'll go through it four times, once per sound. The noise-profile clip itself doesn't get prepped; it's a tool, not a sample.

Set the noise profile (once)

Before the first denoise, Audacity needs to learn what the room noise sounds like. You only do this once; the noise profile stays in memory until you change it or close the program.

  1. Double-click the noise-profile clip (the flat-looking first clip on the track). Double-clicking inside a clip selects that whole clip and nothing else.
  2. Open Effect → Noise Removal and Repair → Noise Reduction…
  3. The Noise Reduction dialog opens. Click the Get Noise Profile button.
  4. The dialog closes immediately. Audacity has captured the noise profile and is now ready to apply the reduction to other clips.

For each of the four sound clips

Now apply the three pipeline steps to each paper-sound clip in turn. Start with paper crumble, slow (the second clip on the track).

1. Denoise.

  1. Double-click the clip to select it.
  2. Open Effect → Noise Removal and Repair → Noise Reduction…
  3. The dialog opens with three sliders: Noise reduction (dB), Sensitivity, Frequency smoothing (bands). Start with 6, 6.00, 6 (a gentle setting that doesn't over-process). Adjust later by ear if needed.
  4. Click OK. Audacity processes the clip and subtracts the noise profile from it. The clip's waveform may look slightly thinner near the silent portions; the body of the sound should look mostly unchanged.

2. Trim.

  1. The clip still has the one-second silence buffer at the start and end. Trim them away now: the buffer was there for headroom during recording; the finished sample shouldn't carry empty silence at either end.
  2. With the selection tool active (the I-beam), click-and-drag to select just the sound itself (from where the sound starts to where it ends, leaving the buffer silence outside the selection).
  3. Apply Edit → Remove Special → Trim Audio, or use the shortcut Cmd + T. This is the same Trim move you used in Module 2: it keeps the selected region and removes everything else on the clip.
  4. The clip now begins where the sound begins and ends where it ends, with no buffer at either edge.
Don't forget the fades

A hard trim almost always cuts the waveform mid-cycle, which produces an audible click at the start or end of the file. Smooth it out with a short fade at each end of the clip:

  1. Zoom into the start of the clip with Cmd + 1 (or Cmd + E after selecting near the edge). 50 ms is too small to select cleanly at the default zoom level; you need to be zoomed in enough to see individual cycles of the wave.
  2. Select about 50 milliseconds at the start of the clip (click-and-drag with the I-beam).
  3. Apply Effect → Fading → Fade In.
  4. Zoom back out with Cmd + F (zoom to fit the whole project).
  5. Repeat at the end of the clip: zoom in, select about 50 ms, apply Effect → Fading → Fade Out, zoom back out.

The Fade In / Fade Out commands and the Cmd + E ↔ Cmd + F zoom pair are all from Module 2; you've used them before.

3. Normalize.

  1. Double-click the trimmed clip to select it.
  2. Open Effect → Volume and Compression → Normalize…
  3. In the dialog, set Normalize peak amplitude to: -1.0 dB.
  4. Make sure Remove DC offset is checked. Leave Normalize stereo channels independently as is (it has no effect on a mono file).
  5. Click OK. Audacity scales the clip's amplitude so the loudest sample sits at -1 dBFS. The waveform expands vertically; quiet bits stay quiet relative to loud bits.
Normalize scales, it doesn't shape

Normalize is a single multiplication applied to every sample. It doesn't change the shape of the sound; it changes its size. A quiet sample stays quiet relative to a loud sample within the same clip. What changes is where the loudest peak sits. Every sample in your library will end up normalized to the same -1 dBFS ceiling, which means you can combine them later without one dominating the mix.

That's one full pipeline run. The clip is now denoised, trimmed (with fades at both ends), and normalized. Repeat for the other three sounds (paper crumble fast, paper rip slow, paper rip fast). Listen to each after you finish to confirm it sounds right.

Step 9 · Export each sample as a WAV

Now you have four prepped clips. Time to export each one as a standalone WAV file into the paper/ folder of your library.

Audacity exports the whole project by default, which would give you one big file with all four sounds in it. You want four separate files, one per sound. The pattern: select a clip first, then export, with the export range set to the current selection.

For each of the four samples:

  1. Double-click the clip to select it.
  2. From the menu bar: File → Export Audio…
  3. The Export Audio dialog opens. Fill it in:
    • File Name: the sample's name following the convention below
    • Folder: click Browse… and pick ~/Documents/lastname/sample-library/paper/
    • Format: WAV (Microsoft)
    • Channels: Mono
    • Sample Rate: 44100 Hz
    • Encoding: Signed 16-bit PCM
    • Export Range: Current selection (this is what limits the export to just the clip you selected; if it's set to Entire Project you'll get all four sounds in one file)
  4. Click Export. If a metadata dialog appears, leave it blank and click OK.

Sample filenames

The naming convention for samples this semester:

[category]-[descriptor]-[variant].wav

Lowercase, hyphens between words, no spaces, no special characters. For today's four sounds:

  1. paper-crumble-slow.wav
  2. paper-crumble-fast.wav
  3. paper-rip-slow.wav
  4. paper-rip-fast.wav

The category is paper: this matches the folder name, which makes the path itself readable (paper/paper-crumble-slow.wav). The descriptor names what the sound is (a crumble, a rip). The variant distinguishes two takes of the same descriptor (slow vs fast).

Step 10 · Write the library README, upload to NAS

A library without a README is a pile of files. A README is the document at the root of the library that explains what's in it, how it's organized, and how to find things. You write the first version today and update it every time you add to the library.

  1. Open TextEdit (in the Applications folder or via Spotlight).
  2. From the menu bar: Format → Make Plain Text (or Cmd + Shift + T). The document is now a plain-text file, not a styled document.
  3. Paste in the template below and fill in the bracketed parts.
  4. Save as README.txt in ~/Documents/lastname/sample-library/.
SAMPLE LIBRARY · [your name]
MUS 381 · Fall 2026

Started: Wed Wk 6, [date]
Last updated: [today's date]

ORGANIZATION

Samples are organized in category folders at the root of the library:
  paper/      Paper sounds (crumbles, rips)

Filename convention:
  [category]-[descriptor]-[variant].wav
  e.g. paper-crumble-slow.wav

All samples are mono, 44.1 kHz, 16-bit WAV.
All samples are prepped: denoised, trimmed, normalized to -1 dBFS.

CONTENTS

paper/  (4 samples)
  paper-crumble-slow.wav    Slow crumble of a sheet of paper, ~8 sec
  paper-crumble-fast.wav    Fast crumble, ~2 sec burst
  paper-rip-slow.wav        Slow tearing, ~7 sec
  paper-rip-fast.wav        Fast tearing, one motion

NOTES

[anything you want to remember about today's session, e.g.
 "the slow crumble took two tries; the first one was too fast"
 or "interesting how different the two rip speeds feel"]
The README is for future-you

The library will grow to dozens of sounds by the end of the semester. By the time you're picking samples for the final project in Module 4, you won't remember what every recording sounds like just from its filename. The README is the document you write to your future self so future-you can find what current-you recorded.
