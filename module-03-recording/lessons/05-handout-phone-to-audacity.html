Step 2 · Open Audacity, pull your library down

Before you import the phone recording, you need your library in front of you so you can put the prepped sample in the right place.

  1. Pull your sample-library/ folder down from the NAS to your local working folder. (Same routine as every Wednesday; the Session Routines card has it. NAS path: students/lastname/sample-library/. Local destination: ~/Documents/lastname/sample-library/.)
  2. Open Audacity. Confirm the project format is still 44100 Hz / 16-bit PCM in Audacity → Preferences → Audio Settings. (It should be from last week, but worth a glance.)
  3. Save a new project file for today's work: File → Save Project → Save Project As…, and save it as sample-library-week-07.aup3 inside ~/Documents/lastname/sample-library/audacity-projects/. Keeping a separate project file per session makes it easy to find your work later.

Step 3 · Import the phone recording

  1. Click anywhere in the empty Audacity project window to make sure it's the active window.
  2. From the menu bar: File → Import → Audio… (or Cmd + Shift + I). Navigate to ~/Documents/lastname/ and select the phone recording. Click Open.
  3. The waveform appears as a new track in the project.
  4. Press spacebar to play it back. Confirm it sounds like what you remember recording. If it doesn't, you probably grabbed the wrong file off the phone; check the filename and try again.

To see the file's properties, open the Audio Track Dropdown Menu: click the three dots (…) next to the track's name in the Track Control Panel (the gray box on the left), or right-click anywhere on the track header. Hover over Format to see the bit depth, and over Rate to see the sample rate. The checked entries are what the track currently is. For a recording made with the recommended phone apps, this will be 48000 Hz and 16-bit PCM or 24-bit PCM. The project's format is 44.1 kHz / 16-bit; the track is at 48 kHz. The two rates don't match.

Stereo or mono?

Phone recordings may be stereo (two channels: left and right) or mono (one channel) depending on the app's pickup-pattern setting. If your track shows two waveform lanes stacked vertically, it's stereo. If it shows one, it's mono. Both work for today; the prep pipeline applies to either. Keep the stereo version if the recording captured spatial information (a hallway, a courtyard, something with left-right movement). Convert to mono if it's a single source recorded up close and the two channels look nearly identical: Tracks → Mix → Mix Stereo Down to Mono. Mono files are half the size and tidier for a sample library.

Step 4 · Resample to 44.1 kHz

The phone recorded at 48 kHz. Your project is at 44.1 kHz. To use the recording in this project (and to keep your library consistent), the two rates need to match. The operation that converts a recording from one sample rate to another is called resampling.

What resampling does

Sample rate is how many measurements the recording stores per second of sound. 48,000 measurements per second is a slightly denser representation than 44,100 per second; both are well above the threshold that captures everything in human hearing. Resampling recalculates the audio at the new rate: Audacity reads the 48 kHz data, runs a math operation that figures out what the same sound would look like at 44.1 kHz, and replaces the track with the recalculated version. The sound stays the same length; the pitch stays the same; what changes is the underlying number of measurements per second.

Resampling is a normal part of recording work. Anytime two devices were running at different sample rates, you need to resample one to match the other. The reason your library standardizes on 44.1 kHz is so that everything in it can be combined and used together without these conversions happening repeatedly down the line.

Resample the imported track

  1. Click anywhere in the Track Control Panel (the gray box on the left of the track). The whole track highlights to show it's selected.
  2. From the menu bar: Tracks → Resample…
  3. The Resample dialog opens with a dropdown labeled New sample rate (Hz). Pick 44100. Click OK.
  4. Audacity processes the track. To confirm the change took effect, open the Audio Track Dropdown Menu again and hover over Rate: 44100 should now be the checked entry. The waveform looks identical; the underlying rate has changed.
Resample vs. project rate vs. export rate

Three different sample rates touch this work, and they're easy to confuse.

  • The track's sample rate is the rate the audio data is stored at on disk. This is what Resample changes.
  • The project's sample rate is the rate Audacity uses for playback and for any new recordings made inside the project. You set this in Preferences (44100 Hz for everything in MUS 381).
  • The export sample rate is the rate of the WAV file Audacity writes when you export. You'll set this to 44100 in Step 6.

If a track and the project don't match, Audacity will play the track back by resampling on the fly (slower playback, often slightly worse quality, and the file on disk is still at the original rate). Running Resample explicitly is the cleaner move: now everything matches, top to bottom.

Step 5 · Run the prep pipeline

The pipeline is the same three steps from Lab 1: denoise → trim → normalize. The one practical difference for a phone recording is the noise profile.

About the noise profile

In Lab 1 you captured a dedicated 10-second noise-profile clip at the top of the project, recorded in the same room with the same mic at the same gain. The denoise step then used that profile to subtract the room's noise floor from the paper recordings.

A phone recording brings its own noise floor with it: whatever was in the room when you hit record, plus the phone's own electronic hiss. So the noise profile has to come from the phone recording itself. The cleanest way: capture the profile from a quiet portion of the recording before or after the sound you actually want.

This is one reason the phone reference card recommends leaving a beat of silence at each end of a phone take. If you didn't, you can still try denoise using the silence buffer Audacity automatically has between clips, but the result will be less consistent. From this session forward, the habit to build: record a second or two of silence at the start of every take, on phone or in the lab. Future-you will use it.

The pipeline

  1. Capture noise profile. Use the I-beam to select a quiet region of the phone recording (a second or two of silence before or after the sound itself). Open Effect → Noise Removal and Repair → Noise Reduction… → click Get Noise Profile. The dialog closes.
  2. Denoise. Double-click the clip to select the whole thing. Open Noise Reduction again. Leave the sliders at 6, 6.00, 6 (a gentle starting point). Click OK.
  3. Trim. Select just the sound itself with the I-beam, leaving the silence outside the selection. Edit → Remove Special → Trim Audio (Cmd + T).
  4. Fade. Zoom in (Cmd + 1) at the start, select about 50 ms, Effect → Fading → Fade In. Zoom out (Cmd + F). Repeat at the end with Fade Out.
  5. Normalize. Double-click the trimmed clip. Effect → Volume and Compression → Normalize… → peak -1.0 dB, Remove DC offset checked, OK.

If any of these steps feels unfamiliar, open Lab 1's handout (Recording into Audacity) and refer to Step 8 there for the full walkthrough with screenshots.

If denoise is making the recording sound underwater

Phone-recorded noise is sometimes harder to subtract cleanly than lab-recorded noise. If the result of denoise sounds hollow, whispery, or has a swimming-pool quality, the profile probably caught some non-noise content (a faint sound, your finger brushing the phone, a passing voice). Undo the denoise (Cmd + Z), pick a different quiet region for the profile, and try again. If no part of the recording is quiet enough, skip the denoise step for this file; trim and normalize will still produce a usable sample, just one that carries its original noise floor.

Step 6 · Export to your library

Decide where the sample lives in your library. The library's organized by category folders at the root: you already have paper/. For today's recordings, you'll probably want to add at least one more category. Pick a folder name that fits what you recorded.

A few example categories, to get you thinking:

  • kitchen/ for a kettle, water, ice, fridge, dishes
  • metal/ for a bike chain, keys, coins, hinges, latches
  • fabric/ for clothing, cushions, bags, anything soft
  • space/ or room/ for ambient recordings of places (a hallway, an elevator, a courtyard)
  • body/ for footsteps, claps, breathing, voice

The category is your call; the names above are starting points. The rule is the same as the paper/ folder: lowercase, hyphenated if needed, no spaces. Pick one that you'll remember and that fits more than just today's sound (so you can add to it later).

  1. In Finder, create the new category folder inside ~/Documents/lastname/sample-library/.
  2. Back in Audacity, double-click the prepped clip to select it.
  3. File → Export Audio… opens the Export dialog. Fill in:
    • File Name: following the convention [category]-[descriptor]-[variant].wav (e.g. kitchen-kettle-boil.wav, metal-keys-jangle.wav)
    • Folder: click Browse… and pick the category folder you just created
    • Format: WAV (Microsoft)
    • Channels: Mono (or Stereo if you kept it stereo in Step 3)
    • Sample Rate: 44100 Hz
    • Encoding: Signed 16-bit PCM
    • Export Range: Current selection
  4. Click Export. Leave the metadata dialog blank.

One sample is now in the library at its new path: for example, ~/Documents/lastname/sample-library/kitchen/kitchen-kettle-boil.wav.

Update the README

Open your library's README.txt in TextEdit. Add the new category to the ORGANIZATION section and the new file to the CONTENTS section. Update the Last updated: date at the top. Save.

The README isn't optional

Every time you add a sample, update the README the same day. The README is the only document in your library that explains what the library is. Skipping it once means you've started a backlog of work you'll never catch up on; updating it as you go takes 30 seconds. By the midterm, the README is part of what gets graded.

Step 7 · Worktime

The rest of the session is yours. A few good ways to spend it:

  • Push more phone recordings through the pipeline. If you have several recordings on your phone, repeat Steps 1 through 6 for each one. Each recording gets its own pass; group them into category folders as you go.
  • Record new sounds in the lab. You have the dynamic mic, interface, and headphones at your station. Use them. Anything in the room is fair game: keys, coins, your own voice, a chair scraping, a water bottle, a notebook flipping. The full record-and-prep procedure from Lab 1 still applies; you already know how to do it.
  • Walk the building. Step out into the hallway, the stairwell, the courtyard with your phone. Record something there. Bring it back, push it through the pipeline.
  • Organize what you have. Look at your library so far. Are the categories making sense? Are filenames clear? Is the README accurate? Library work isn't just adding files; it's also keeping the structure usable as it grows.

The TA is in the room. Ask for help if a recording isn't working, if denoise is producing strange artifacts, if a file isn't where you expect it. This is the one session in the module where the schedule has slack built in for getting unstuck.
