Audacity Orientation

Open the DAW. Import a sound. Make your first edit. Save it the right way.

Audacity is the DAW we'll use for the rest of Module 2. By the end of this lab session, you'll have opened it, set its project format to the course standard, imported a short sample from the NAS, played it back, made a basic edit, exported the result, and uploaded everything back to the NAS so it's there for you next session. None of these moves is hard. The point of doing them now is so the workflow is automatic by the time Project 1 starts.

Today's gear

Take from the lab's gear storage: audio interface, headphones. Plug in and run through the start-of-session steps on the laminated Session Routines card before continuing here.

What we're doing today

  1. Pull the orientation sample from the NAS into your local working folder.
  2. Open Audacity and set the project format (44.1 kHz, 16-bit).
  3. Tour the Audacity interface: the parts you'll use today and the names of things.
  4. Import the sample, play it back, listen.
  5. Make one selection, one cut, one fade. Listen to the difference.
  6. Save the project, then export to WAV at the course-standard format.
  7. Upload everything back to the NAS using the end-of-session routine.
Before you start

Today is the first session where you run the full start-of-session routine yourself, from scratch. It will feel slow today; by Wk 4 it will be muscle memory. Keep the laminated Session Routines card visible so you can refer to it as we go. At the end of class you'll do the same routine in reverse.

Step 1 · Pull the orientation sample from the NAS

Today's sample lives on the class NAS in shared/module-02/orientation/. It's a short bell-like resonance recorded in stereo, about sixteen seconds long, with a dense ringing body that slowly fades to silence. You'll use it to learn Audacity's basic moves.

You're going to grab the sample during your start-of-session routine, before the NAS gets ejected. If you're already past the NAS phase of the routine and the NAS is no longer in Finder's sidebar, reconnect (Cmd + K, server address, sign in).

  1. With the NAS mounted, navigate in Finder to shared/module-02/orientation/.
  2. You'll see one file: orientation-sample.wav. Copy it (drag it while holding Option, or right-click and choose Copy) into your local folder at ~/Documents/lastname/orientation/. Make an orientation folder if you don't have one yet.
  3. Verify the copy is there. Don't open it from the NAS: always work from your local folder.
  4. Now eject the NAS (Finder sidebar, eject icon next to the NAS name). You're done with NAS for the working portion of the session.
Local-first, NAS as sync

You always work from your local folder, never directly from the NAS. The NAS is the master copy that travels between you and the next session; your local folder is your live workbench. Pull from NAS at the start of session, work locally, push back to NAS at the end. This is the same workflow on the laminated routines card: internalize it now.

Step 2 · Open Audacity, set the project format

  1. In the Dock, find the Audacity icon (a pair of orange headphones around a sound wave). Click it once.
  2. Audacity opens with an empty project window. Don't import anything yet.
  3. From the menu bar, open Audacity → Settings… (or Cmd + ,).
  4. Click Audio Settings (or, in older versions of Audacity, Quality).
  5. Set Project Sample Rate to 44100 Hz.
  6. Set Default Sample Format to 16-bit PCM.
  7. Close the Settings window. The project is now configured to the Module 2 course standard.
Why 44.1 kHz / 16-bit?

This is the CD standard, the format Schaeffer's recordings sit at on a remaster, and what we agreed on for Module 2 in Monday's reading. In Module 4 we'll switch to Ableton at 48 kHz / 32-bit. For now, every project you make in Audacity uses these numbers.

Step 3 · A tour of the interface

Before you import anything, take a minute to learn the names of the things on screen. You don't need to memorize all of these, just be able to find them when the instruction says "click the play button" or "use the selection tool."

Two things to know for today:

  • The Selection tool (the I-beam icon in the Tools toolbar) is what you use to click and drag a region of audio. We'll use it for every edit. Make sure it's selected.
  • The play button in the Transport toolbar plays the project. Spacebar does the same thing, and it's faster. Use the spacebar.

Step 4 · Import the sample, play it, listen

  1. From the menu bar, click File → Import → Audio… (or Cmd + Shift + I).
  2. Navigate to your local folder at ~/Documents/lastname/orientation/ and select orientation-sample.wav. Click Open.
  3. The waveform appears as a stereo track with two lanes (left channel on top, right channel below). The dense ringing body of the sound fills the first half; the second half tapers as the resonance decays toward silence.
  4. Press spacebar to play. Listen. Press spacebar again to stop.

Look at the waveform shape. You can see the envelope from Monday's reading right there: a sustained body with internal swells, then a long release as the resonance fades out. The picture is the sound. Make a habit of looking at the waveform as you work; it tells you what you're going to hear before you hear it. Notice also that the two lanes (left and right channels) are similar but not identical: that's what makes this sample stereo rather than mono.

Step 5 · Your first edit: select, cut, fade

Now make a deliberate edit. The goal: shorten the sample's decay so it ends sooner. We'll do three small moves (selection, cut, fade-out) and listen to each.

Make a selection

  1. Make sure the Selection tool (the I-beam) is active in the Tools toolbar.
  2. Click and drag in the waveform from somewhere in the middle of the decay (around 7 seconds in) to the very end of the file. Drag across the full track height so the selection covers both stereo lanes. The selected region appears highlighted across both channels.
  3. Press spacebar. Audacity plays only the selection. You'll hear just the tail end of the decay.

Cut it

  1. With the selection still highlighted, press Cmd + X (or Edit → Cut).
  2. The selected region disappears from the track. The sample now ends where you cut.
  3. Press Cmd + Home to jump to the start, then spacebar to play. Listen. The sample now stops abruptly: an unnaturally hard release.

Soften the new ending with a fade

An abrupt cut at the end of an audio clip usually produces a click pop, because the waveform doesn't end at zero. A short fade-out fixes that and makes the ending more graceful.

  1. Click somewhere in the last couple of seconds of audio, then drag a selection covering the final ~2 seconds across both stereo lanes. (Watch the selection toolbar at the bottom: it shows start and end times to the millisecond.)
  2. From the menu bar, click Effect → Fading → Fade Out.
  3. The waveform in the selected region now ramps down to silence visually on both channels. Play the file again. The cut-off ending now fades cleanly instead of stopping hard.
What you just did

Three operations: selection, cut, fade. These are the same three primitives you'll use for the rest of the course. Mon Wk 3's reading expands the editing vocabulary to a dozen moves; all of them are variations on selecting a region and doing something to it.

Step 6 · Save the project, then export to WAV

Audacity has two distinct save operations and you need to use both. They do different things.

  • Save Project writes Audacity's working file (an .aup3 file). This preserves all your tracks, edits, undo history, and project settings. You can re-open it and keep working. Other software cannot read it.
  • Export writes a finished audio file (a .wav) that any music software, media player, or upload form can open. This is what you submit, share, or play outside of Audacity.

You'll do both: save the project so you can come back to it next session, and export the WAV so the finished sound exists as a real file.

Save the project

  1. File → Save Project As… (or Cmd + Shift + S).
  2. Navigate to ~/Documents/lastname/orientation/.
  3. Name the file lastname-orientation.aup3. (Replace lastname with your actual last name, lowercase.) Click Save.

Export to WAV

  1. File → Export Audio… (or Cmd + Shift + E).
  2. The first time you export, Audacity shows a "How would you like to export?" prompt with two options: Share to audio.com (a cloud service) and On your computer (a local file). You want the file saved locally on your machine. Click Export to computer. While you're here, tick Don't show again in the bottom-left so this prompt doesn't appear every time you export.
  1. The Export Audio dialog opens. Fill it in:
    • File Name: lastname-orientation.wav (lowercase, with your last name)
    • Folder: click Browse… and pick ~/Documents/lastname/orientation/
    • Format: WAV (Microsoft)
    • Channels: Stereo (matches the source)
    • Sample Rate: 44100 Hz
    • Encoding: Signed 16-bit PCM
    • Export Range: Entire Project
  2. Click Export. If a metadata dialog appears, you can leave it blank and click OK.
Filename conventions

Lowercase only. Hyphens between words. No spaces, no apostrophes, no accented characters. your-last-name-orientation.wav is correct; YourLastName Orientation Final.wav will cause problems with the NAS, with command-line tools, and eventually with you. Build the habit now.
