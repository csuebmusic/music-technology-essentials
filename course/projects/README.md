# Projects

Project prompts and rubrics for MUS 381.

## Format

All project prompts are HTML, using the central stylesheet at [`assets/style.css`](../../assets/style.css). They share the same visual system as handouts, readings, and listening assignments.

Filename: `project-XX-shortname.html` (e.g. `project-01-musique-concrete.html`).

CSS link path: `../../assets/style.css`.

## Standard structure

Every project prompt should include:

- **The brief** — what the student is building, in a few sentences
- **Constraints** — length, source material, software, format, naming
- **What "demonstrating skills" means** — the qualitative goals
- **Process expectations** — start early, save versions, etc.
- **Presentation/submission details** — when and where
- **Rubric** — explicit criteria with point values
- **What I'm not grading** — equally important to set expectations
- **Late policy**
- **Drawing inspiration** — connection back to listening, references, examples

## Tone

Student-facing. Direct about expectations. Generous about creative latitude. The goal is for students to leave the prompt knowing exactly what they have to do, what they have freedom to do, and what they'll be evaluated on.
